#### 3. Age Malus: −15 Punkte für alte Posts ohne Qualitätssignal

Posts mit `ageMinutes > 60` UND keinem der folgenden Qualitätssignale:
- `whaleCount >= 1` (mindestens ein Whale-Kurator)
- `authorAvgSpPerVp >= 0.10` (überdurchschnittliche Author-History)
- Kategorie `immer_voten` oder `lieblingsautor`

→ erhalten −15 Punkte auf den `finalScore` (nicht auf `score`, der Rohwert bleibt sichtbar).

Kein hartes Skip-Gate: alte Posts können trotzdem durchkommen, wenn andere Signale sie tragen. Die Simulation hatte gezeigt, dass ein hartes Gate (>60 min → skip) 1144 TPs blockiert hätte und nur 215 FPs verhindert — weil 83% aller Shadow-Run-Zeilen Wiederholungs-Evaluierungen von Posts >24h sind.

## Architektur-Entscheidung: WhaleQuality (Roadmap Phase 3+)

### Zwei orthogonale Signale — kein Ersatz, sondern Ergänzung

- **whaleCount** → "Wie viel Aufmerksamkeit bekommt der Autor?" (Quantität, bleibt valide)
- **whaleQuality** → "Wie gut sind die Leute, die ihm Aufmerksamkeit schenken?" (Qualität, neu)

`whaleCount` zählt Kuratoren mit genug SP — unabhängig von ihrer Discovery-Qualität.
Kafio-Analyse zeigt die Lücke: 16 "Whale-Votes" = 3 Self-Votes + tipu (Bulk-Voter, 74% Precision, Ø 2.4 SBD auf kafio-Posts) + xpilar-Cluster. Kein echter Discovery-Kurator — trotzdem passiert Gate 3, weil whaleCount > 0.

### Datenbasis (steht bereit)

`vb_whale_vote_details`: 688k Vote-Events, 250 Kuratoren, März–Juni 2026.

Discovery-Precision-Analyse (Early-Votes ≤ 20 min, Payout-Threshold ≥ 2 SBD, min. 30 resolved):

| Kurator | Discovery-Precision | Ø Payout (frühe Votes) | Votes gesamt |
|---|---|---|---|
| oppps | 100% | 32.75 SBD | 3.357 |
| ekatirina | 100% | 57.96 SBD | 1.212 |
| steeminate | 97% | 26.44 SBD | 3.376 |
| cur8 | 91.5% | 23.54 SBD | 6.148 |
| tipu | 74.3% | 10.02 SBD | 26.239 |

### Ziel-Architektur

```
vb_curator_quality        ← täglich aus vb_whale_vote_details berechnet
  curator_name
  early_votes_resolved
  discovery_precision     (P(payout >= 2 SBD | voted ≤ 20 min))
  avg_payout_early
  quality_score           (0–10, composite)
  quality_tier            (elite / good / moderate / low / bot)
  computed_at

whaleQuality = Σ(quality_score × min(sp_weight, cap)) für frühe Voter eines Posts
             — Self-Votes ausgeschlossen
             — Known-Bulk-Voter (tipu, jsup) auf 0.2× Gewicht gedeckelt
```

### Erwarteter Effekt

```
kafio-Post:  whaleCount=16 → whaleQuality≈0.8   ← Self-Votes + Bulk-Voter
Top-Post:    whaleCount= 2 → whaleQuality≈12.4  ← oppps + steeminate
```

Gate 3 würde dann auf `whaleQuality < Schwellwert` prüfen statt `whaleCount === 0`.
Auch `signalCurators`-Komponente im OpportunityScore profitiert direkt.
