# DevLog — 5. Juni 2026

## Vote-History Regression — testuser1 / Accounts mit kurzer Historie

**Entdeckt durch:** Erster externer Tester [@michelangelo3](https://steemit.com/@michelangelo3) beim Einstiegstest mit Account `testuser1`.

**Symptom:** "Fehler beim Laden der Vote-Historie" (roter Fehler, blockiert gesamten DNA-Tab) für Accounts mit weniger als ~600 Gesamtoperationen in der Chain-History.

**Root Cause:**

Die Steem API `get_account_history` verlangt `start > limit` (strikt). Der Batch-Loop in `voteHistory.ts` sendete immer `limit = MAX_OPS_PER_BATCH (100)`. Wenn ein Account wenige Gesamtoperationen hat, fällt `fromSeq` in der letzten Iteration unter 100 — z.B. `fromSeq = 66, limit = 100` → `66 < 100` → API-Fehler.

`testuser1`: 571 Gesamtops → letzte Iteration `fromSeq=66, limit=100` → API wirft `"args.start >= args.limit: start must be greater than limit"`.

**Fix:** `apps/api/src/chain/voteHistory.ts`
```typescript
const limit = fromSeq < 0
  ? MAX_OPS_PER_BATCH
  : Math.min(Math.max(fromSeq - 1, 0), MAX_OPS_PER_BATCH);
if (fromSeq >= 0 && limit <= 0) break;
```

Garantiert `start > limit` auf allen Calls. Commit: `8534250`.

**Deployment-Pfad-Fehler entdeckt:** Während der Fehlersuche stellte sich heraus, dass `docker cp ... votebroker-modern-api-1:/app/dist/` an die falsche Stelle kopiert. Der Node.js-Prozess läuft aus `/app/apps/api/dist/`, nicht `/app/dist/`. Korrekter Befehl:

```bash
docker cp apps/api/dist/. votebroker-modern-api-1:/app/apps/api/dist/
docker restart votebroker-modern-api-1
```

Architektur-Dokumentation in `docs/architecture/overview.md` korrigiert.
